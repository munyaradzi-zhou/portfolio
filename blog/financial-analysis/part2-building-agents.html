The Tools Layer: Three Capabilities Every Agent Can Use

Tool 1: Tavily Search — The Eyes of the System

Tavily is an AI-optimized search API that returns results with relevance scores. Unlike raw Google search, Tavily's results are pre-filtered for quality and come with extracted content — no HTML parsing needed.

async def tavily_search(query: str, max_results: int = 5) -> list[dict]:
    client = AsyncTavilyClient(api_key=settings.tavily_api_key)
    response = await client.search(
        query=query,
        max_results=max_results,
        search_depth="advanced",     # Highest quality results
        include_answer=False,        # We synthesize our own answers
    )
    return response.get("results", [])

The search_depth="advanced" flag is critical. In standard mode, Tavily returns basic snippets. In advanced mode, it extracts full article content — the difference between "Apple reported Q4 earnings" and a 500-word excerpt with specific revenue figures, EPS, and segment breakdowns.

Tool 2: Python REPL — The Calculator

Agents sometimes need to compute things: moving averages, percentage changes, valuation ratios. The Python REPL provides a sandboxed execution environment with restricted builtins:

ALLOWED_BUILTINS = {
    "abs", "all", "any", "bool", "dict", "float", "int", "len", "list",
    "max", "min", "print", "range", "round", "set", "sorted", "str", "sum", "tuple", "zip",
}
SAFE_MODULES = {"math", "statistics", "json", "datetime", "collections", "itertools"}

Any attempt to import os, subprocess, sys, or anything outside the safe list is immediately blocked. This is important for a system that executes LLM-generated code.

Tool 3: Azure Cosmos DB Memory — The Brain

Every time the Researcher finishes a task, it stores its findings in Cosmos DB with a 1536-dimensional embedding vector. The next time a related query comes in, the Researcher checks memory first before searching the web.

class MemoryStore:
    async def store(self, content: str, metadata: dict) -> str:
        vector = await self._embed(content)          # text-embedding-3-small
        document = {
            "id": str(uuid.uuid4()),
            "content": content,
            "embedding": vector,                      # 1536 floats
            "type": metadata.get("type", "research"),
            "created_at": datetime.now(timezone.utc).isoformat(),
            **metadata,
        }
        self._container.create_item(body=document)
        return document["id"]

    async def recall(self, query: str, top_k: int = 5) -> list[dict]:
        query_vector = await self._embed(query)
        items = list(self._container.query_items(
            query="SELECT c.id, c.content, c.embedding, c.type, c.task "
                  "FROM c WHERE c.embedding != null",
            enable_cross_partition_query=True,
        ))
        # Client-side cosine similarity
        scored = []
        for item in items:
            score = cosine_similarity(query_vector, item["embedding"])
            scored.append((score, item))
        scored.sort(key=lambda x: x[0], reverse=True)
        return [{"content": item["content"], "score": score}
                for score, item in scored[:top_k]]

In our live test, the memory store held 9 documents and recall took 1.36 seconds. The cost? 7.71 RUs for the query (~$0.00002).

Agent 1: The Researcher — Multi-Pass Financial Web Search

The Researcher is the most complex agent. It doesn't just run one search — it generates targeted sub-queries, runs them in parallel, deduplicates results, and synthesizes everything with source attribution.

Step 1: Check Memory

memory = MemoryStore()
prior_research = await memory.recall(task, top_k=3)

# In our Apple test, this returned 3 memories with score 0.648:
# "Apple Inc. (NASDAQ:AAPL) reported Q3 2023 revenue of $81.8 billion..."

Step 2: Generate Sub-Queries

A single search query like "Analyze Apple stock" is too broad. The Researcher uses GPT-4o-mini (temperature 0.0) to break it into 3 targeted queries covering fundamentals, market trends, and risk factors.

Live output:

[
  "Apple Inc. financials Q3 2023 earnings revenue metrics analysis",
  "Apple stock market trends sector analysis competitive landscape 2023",
  "Apple Inc. risk factors analyst opinions recent news October 2023"
]

Step 3: Parallel Search

search_tasks = [tavily_search(q, max_results=8) for q in sub_queries]
all_results = await asyncio.gather(*search_tasks)
# 3 x 8 = 24 results → 22 unique after URL deduplication

Three targeted queries from different angles produce 22 unique sources in the same time a single query would take. The parallel execution means the wall-clock time is just the slowest single search (~4 seconds for all three).

Step 4: Deduplicate and Synthesize

After deduplication by URL, all sources are passed to GPT-4o-mini for synthesis into comprehensive financial research notes with specific numbers and source URLs.

Step 5: Store to Memory

The first 2,000 characters of the research output get stored in Cosmos DB for future sessions.

Total Researcher time: 23.43 seconds — memory recall (5.2s) + sub-query generation (2.3s) + parallel search (4.0s) + LLM synthesis (11.0s) + memory store (1.5s).

Agent 2: The Analyst — Structured Data Extraction

The Analyst takes the Researcher's free-text notes and extracts a rigid JSON structure. Temperature: 0.0 — deterministic extraction.

The Prompt: Be Specific About Structure

The key to reliable JSON extraction is showing the exact schema you want:

{
    "key_metrics": [
        {"metric": "P/E Ratio", "value": "28.5x", "source": "URL"},
        {"metric": "Market Cap", "value": "$3.2T", "source": "URL"}
    ],
    "market_trends": [
        {"trend": "...", "impact": "positive/negative/neutral", "evidence": "...", "source": "URL"}
    ],
    "risk_factors": [
        {"risk": "...", "severity": "high/medium/low", "details": "...", "source": "URL"}
    ],
    "competitive_position": {
        "market_share": "...",
        "key_competitors": ["..."],
        "competitive_advantages": ["..."],
        "competitive_risks": ["..."]
    },
    "financial_health": {
        "profitability": "strong/moderate/weak",
        "growth_trajectory": "accelerating/stable/decelerating",
        "balance_sheet": "strong/moderate/weak",
        "cash_flow": "strong/moderate/weak"
    }
}

The critical constraint: key_metrics must contain at least 5 entries.

Agent 3: The Writer — Professional Report Generation

The Writer transforms structured data into a professional investment analysis report. Temperature: 0.3 for natural writing without unpredictable financial claims.

The Prompt: Enforce Structure, Not Words

Instructions:
- CRITICAL: Every section MUST contain at least 2 inline citations [Source: URL]
- CRITICAL: Every section MUST include specific financial numbers and metrics
- Include standard investment disclaimers
- Present both bull and bear cases objectively
- Never make unqualified buy/sell recommendations

Handling Revisions

The Writer's most important capability is handling Critic feedback. On revision cycles, it receives the previous critique and must make targeted improvements:

if critique and revision_count > 0:
    prompt_parts.append(
        f"## Revision Feedback from Critic (Revision #{revision_count})\n"
        f"{critique}\n\n"
        "Address ALL the revision instructions above. Make targeted improvements "
        "while preserving the parts that were already good.\n\n"
    )

Without "preserve the parts that were already good," the Writer tends to rewrite everything from scratch, losing improvements from the previous draft.

Agent 4: The Critic — Calibrated Quality Scoring

The Critic is the most carefully engineered agent. A bad Critic breaks the system in two ways: too lenient (approves garbage) or too strict (infinite revision loops).

Calibrated Scoring Rubrics

1. Accuracy (0-10):
   9-10: Every financial claim has a citation, all metrics properly sourced
   7-8:  Most claims cited, minor data gaps
   5-6:  Multiple unsourced financial claims

2. Completeness (0-10):
   9-10: Covers fundamentals, technicals, competitive landscape, risks, thesis
   7-8:  Covers main aspects, some areas could be deeper
   5-6:  Major financial aspects missing entirely

3. Risk Disclosure (0-10):
   9-10: Comprehensive risk section, proper disclaimers, balanced bull/bear
   7-8:  Adequate risk coverage, disclaimer present
   5-6:  Missing disclaimers or one-sided analysis

4. Clarity (0-10):
   9-10: Excellent structure, professional tone, actionable insights
   7-8:  Good structure, minor readability issues
   5-6:  Disorganized or unclear

The Calibration Anchor

The most important line in the Critic's prompt:

Calibration guidance:
- A report with proper disclaimer, financial metrics, and risk section deserves >= 7
- Do NOT penalize for lacking tangential information not directly asked about
- Focus revision instructions on the 1-2 most impactful improvements

Without this anchor, GPT-4o-mini tends to give 6/10 scores to perfectly adequate reports and generate 5+ revision instructions per cycle. The calibration guidance ensures consistent scoring.

Max Revisions Safety Valve

MAX_REVISIONS = 3

if revision_count >= MAX_REVISIONS and critique.get("status") == "REVISE":
    critique["status"] = "APPROVED"
    critique["feedback"] += " [Auto-approved: max revision limit reached]"

This prevents infinite loops. In practice, most reports reach 8+ scores by revision 2–3, so the safety valve rarely triggers.

Prompt Engineering: Lessons from Financial Analysis

Lesson 1: Show, Don't Tell. Bad prompt: "Extract financial metrics." Good prompt: Shows the exact JSON schema with example values.

Lesson 2: Separate Concerns Ruthlessly. The Researcher gathers data. The Analyst extracts numbers. The Writer writes. No agent's prompt mentions another agent's job.

Lesson 3: Calibrate the Critic Explicitly. Without calibration guidance, the Critic gives wildly inconsistent scores. Adding explicit score ranges and anchoring made scoring consistent across runs.

Lesson 4: "Preserve What's Good" on Revisions. Without this instruction, the Writer rewrites from scratch on each revision, losing improvements from the previous draft.
